Version Control is important because it helps a programming team keep track of the possible changes that may have occured. This allows developers to go back and look at previous version in case a mistake occurs. Disruption amongst team members is also diminished while searching for a mistake.

Source 1

Git is a locally installed version control tool that assists in managing the history of a source code. GitHub, on the other hand, is a cloud based service that stores code used by the Git tool. It also allows for easier collaboration on projects.

Steps to set up Git

  1. Open a terminal and run "sudo apt update"
  2. Use the command "sudo apt install git" to install Git
  3. Check the version of Git you have by typing "git --version"
  4. Next, you will configure your Git username by typing the command "git config --global user.name "(enter username)""
  5. To congigure your email, use this command: "git config --global user.email "(enter email address)""
  6. Initialize Git into the file structure you'd like by typing "git init" into the terminal
